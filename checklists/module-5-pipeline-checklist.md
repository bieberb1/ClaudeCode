# Module 5: Pipeline Verification Checklist

Use this checklist at every stage and every handoff. Verification is not a single gate at the beginning — it runs through the entire pipeline.

---

## Stage 1: Research

Before moving to Stage 2, confirm:

- [ ] I identified the primary source material (keynote, report, dataset, interview)
- [ ] I ran every key statistic through Perplexity or a named primary database
- [ ] For each claim, I recorded: the original assertion, the current verified figure, the source name, and today's date
- [ ] I noted which claims are Confirmed, Updated, Revised, or Contested
- [ ] I did not use any figure whose original source I cannot name
- [ ] I did not rely on LLM-generated numbers without tracing them to a named source
- [ ] I flagged any claims that are temporally fragile (carbon budgets, annual counts, ongoing research)

---

## Stage 2: Structure (Ground Truth Document)

Before moving to Stage 3, confirm:

- [ ] Every claim in the Ground Truth Document has passed Stage 1 verification
- [ ] Each entry has: claim, verified figure, source, date, thread tag, status
- [ ] I identified my hero metric (the single most compelling number for my chosen audience)
- [ ] I identified at least one inciting incident (the surprising finding that drives narrative urgency)
- [ ] Claims are tagged by narrative thread so I can pull the right evidence per audience
- [ ] I have not included any unverified claims in the document

---

## Stage 3: Narrate

Before moving to Stage 4, confirm:

- [ ] I defined the target audience before writing the Story Spine
- [ ] I applied the Module 2 audience diagnostic (Who are they? What do they already value? What frame works?)
- [ ] The Story Spine uses only evidence from the Ground Truth Document
- [ ] The emotional register is calibrated to the audience (not just to the source material)
- [ ] The value frame is matched to what this audience already cares about
- [ ] The causal chain in "Because of that / Because of that" is logically sound
- [ ] The resolution is credible and audience-appropriate
- [ ] I have not introduced any claims in the narrative that are not in the Ground Truth

---

## Stage 4: Produce

Before moving to Stage 5, confirm:

- [ ] I selected tools by audience, channel, and stakes (not by novelty)
- [ ] Every statistic in the finished deliverable traces back to a Ground Truth entry
- [ ] Visualizations accurately represent the underlying data (no misleading axes, no cherry-picked ranges)
- [ ] Any AI-generated visuals or text have been reviewed for accuracy against the Ground Truth
- [ ] The emotional tone of the finished piece matches the calibration from Stage 3
- [ ] I checked any auto-generated text (Gamma slide notes, AI narration scripts) against the Ground Truth

---

## Stage 5: Multiply

Before distributing, confirm:

- [ ] Each derivative output has been reviewed independently, not just assumed to be correct
- [ ] Shorter formats (social cards, one-pagers) still accurately represent the underlying evidence
- [ ] Adapted frames for new audiences use the same verified data, not new unverified claims
- [ ] If the data has been updated since Stage 1, the Ground Truth has been updated and all downstream outputs reviewed
- [ ] Localized or audience-specific versions have been checked to ensure they do not misrepresent the evidence for that context

---

## At every handoff

Ask at each transition between stages:

1. **Source integrity:** Can every fact in this output be traced to the Ground Truth Document?
2. **Currency:** Are any figures in this output potentially outdated since Stage 1 verification?
3. **Framing accuracy:** Does the framing accurately represent the evidence, or has emphasis drifted?
4. **Audience fit:** Is the emotional register still calibrated for the intended audience?
5. **AI output check:** Has any AI-generated content been reviewed by a human against the source evidence?
